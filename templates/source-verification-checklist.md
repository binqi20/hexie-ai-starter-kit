# Source Verification Checklist

Use this before putting an AI-assisted claim into coursework, a proposal, or a presentation.

| Check | Yes/No | Notes |
|---|---|---|
| I opened the original source. |  |  |
| I recorded source status. |  | opened_full_text / metadata_only / secondary_summary / not_checked |
| The citation is real. |  |  |
| The cited passage supports the claim. |  |  |
| I recorded an evidence anchor. |  | page / section / table / figure / paragraph / short quote |
| I assigned support status. |  | SUPPORTED / PARTIAL / UNSUPPORTED / CONTRADICTED / NOT_CHECKED |
| I checked the method or data behind the claim. |  |  |
| I recorded any limitation or uncertainty. |  |  |
| I did not rely only on an AI summary. |  |  |
| I removed or rewrote unsupported claims. |  |  |

## Final question

Would I be comfortable explaining this source and claim to my instructor?

## Support-status rule

- Use `SUPPORTED` for final claims.
- Use `PARTIAL` only after narrowing the wording.
- Remove `UNSUPPORTED` and `CONTRADICTED`.
- Do not submit or present `NOT_CHECKED` claims as evidence.
