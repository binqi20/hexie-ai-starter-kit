# Workflow: Literature Map

Use this workflow to turn a small set of readings into a clear map of themes, theories, methods, and open questions.

## Best for

- Early dissertation planning.
- Seminar preparation.
- Comparing five to ten papers.
- Building an evidence map for a management topic.

## Inputs

- A research question.
- Five to ten checked sources.
- One research note per source.
- `templates/claim-evidence-matrix.md`.

## Steps

1. Define the map question.
   - Example: "How do family firms manage succession under digital transformation?"

2. Group notes by contribution type.
   - Theory or concept.
   - Method or data.
   - Empirical finding.
   - Boundary condition.
   - Practical implication.

3. Build a comparison table.

| Source | Context | Theory or lens | Method | Main finding | Limitation | Useful anchor |
|---|---|---|---|---|---|---|
| Source 1 |  |  |  |  |  |  |
| Source 2 |  |  |  |  |  |  |

4. Identify tensions.
   - Do sources agree?
   - Do they study different contexts?
   - Are findings based on different methods?
   - Are there hidden assumptions?

5. Write a synthesis claim.
   - Use only claims with evidence anchors.
   - Mark the confidence level: high, medium, low.

6. Create next-search questions.
   - What evidence is missing?
   - Which keyword, theory, context, or method should be searched next?

## AI prompt

Use `prompts/literature-synthesis-from-notes.md`.

## Human gate

Before using the map in writing, check that each synthesis claim is supported by at least two anchors or clearly labeled as a single-source observation.
